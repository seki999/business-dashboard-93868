# AI 技術検証計画

## 検証目的

機械設備センサーデータに対する異常検知モデルの適用可能性を確認し、AI システム開発前の技術的課題を洗い出す。

## 検証範囲

- 模擬センサーデータ生成
- 欠損値処理と標準化
- IsolationForest による異常検知
- FastAPI 推論 API
- Streamlit Dashboard
- 検証レポート自動生成

## 評価観点

- 異常データを検出できるか
- 推論結果に理由を付与できるか
- API と Dashboard で検証結果を確認できるか
- Linux / Docker 環境で再現可能か
- 技術資料として第三者に説明可能か

## 成果物

- ソースコード
- Dockerfile / docker-compose.yml
- README
- 技術検証レポート
- docs 配下の設計・調査資料

## リスク

- 模擬データと実機データの分布差
- 異常ラベルの定義不足
- 設備ごとの個体差
- 運用時のモデル劣化

## 次ステップ

実機データを入手し、特徴量設計、閾値設計、モデル比較、運用監視設計を追加検証する。
