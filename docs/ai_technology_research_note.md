# AI 技術調査ノート

## 調査目的

機械設備のセンサーデータに対して、AI による異常検知を適用する際の候補技術と PoC での確認観点を整理する。

## 候補技術

### IsolationForest

木構造を用いて外れ値を検出する手法。異常データが少ない初期検証に適している。モデル構造が比較的軽量で、API 化もしやすい。

### One-Class SVM

正常データの境界を学習する手法。特徴量スケールに敏感であり、標準化やパラメータ調整が重要になる。

### AutoEncoder

ニューラルネットワークで再構成誤差を用いる手法。非線形な異常パターンに対応しやすいが、学習データ量と運用設計が必要になる。

### 時系列モデル

LSTM や Transformer 系のモデルを用いることで、センサー値の時間的変化を考慮できる。故障予兆検知では有効だが、PoC 初期では実装・評価コストが高い。

## 今回の採用方針

初期 PoC では IsolationForest を採用する。理由は、実装が軽量で、異常データが十分にない状況でも技術検証を始めやすく、FastAPI / Streamlit との連携も容易なためである。
